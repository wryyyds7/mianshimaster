# 功能文档 — 面试大师

## 一、产品概述

面试大师是一款运行于PC端的AI辅助问答软件，核心设计原则是**提问者与回答者身份绝对隔离**。系统支持双模式运行（本地API / 服务器API），并预留APP扩展能力。

## 二、用户角色定义

| 角色 | 权限 | 说明 |
|------|------|------|
| **提问者** | 提出问题、查看回答 | 匿名或登录，身份不可见 |
| **回答者** | 启动回答工作台、管理知识库、查看历史 | 需登录/配置API后方可使用 |
| **管理员** | 系统配置、用户管理（服务器模式） | 可选 |

**身份隔离规则**：
- 同一物理用户在同一会话中，身份固定（回答者模式中不会看到提问者信息）
- 提问者不暴露任何个人信息给回答者
- 大模型回答时不携带提问者身份标记

## 三、功能模块详述

### 3.1 双模式访问与配置

```
┌─────────────────────────────────────┐
│           启动应用                   │
└─────────────┬───────────────────────┘
              │
              ▼
     ┌───检测API配置状态───┐
     │                    │
  [已配置]            [未配置]
     │                    │
     ▼                    ▼
  直接使用          弹窗提示：
  本地API          "请配置好您的API信息
  (优先)              或者进行登录"
                     │
                     ▼
              ┌──用户选择──┐
              │           │
          [配置API]    [登录服务器]
              │           │
              ▼           ▼
         填写API-Key   输入账号密码
              │           │
              └─────┬─────┘
                    ▼
              进入主界面
```

**配置项**：
- **API模式**：自定义API Key（支持OpenAI兼容接口）
- **服务器模式**：连接远程服务器，使用服务器端配置的AI
- **切换规则**：用户自定义API > 服务器API；可在设置中随时切换

### 3.2 回答辅助工作台

#### 3.2.1 进入流程
1. 用户上传背景文件（PDF/Word/TXT/Markdown）
2. 上传的文件内容作为"上下文"优先传递给大模型
3. 点击"开始回答" → 进入回答者工作台
4. 身份锁定为"回答者"

#### 3.2.2 工作台界面布局

```
┌──────────────┬──────────────────────────────┐
│  问题列表    │       对话详情区               │
│  (左侧30%)   │       (右侧70%)               │
│              │                              │
│ ┌──────────┐ │  ┌────────────────────────┐  │
│ │问题标题1  │ │  │ 提问者：问题内容...     │  │
│ │2024-01-15│ │  │ 大模型：回答内容...     │  │
│ └──────────┘ │  └────────────────────────┘  │
│ ┌──────────┐ │                              │
│ │问题标题2  │ │  ┌────────────────────────┐  │
│ │2024-01-15│ │  │ 输入框                │  │
│ └──────────┘ │  │ [发送]                │  │
│              │  └────────────────────────┘  │
│ [退出]按钮   │                              │
└──────────────┴──────────────────────────────┘
```

**核心规则**：
- **新问题优先**：当提问者提出新问题时，大模型立即生成回答，优先级最高（打断当前操作）
- 左侧问题列表以"问题标题"为索引
- 点击历史问题可查看完整对话
- 设有明确的"退出"按钮，点击即退出回答者身份

#### 3.2.3 背景文件管理
- 支持拖拽/点击上传
- 支持格式：PDF、Word(.docx)、TXT、Markdown(.md)
- 上传后自动解析文本内容
- 提取的文本作为"系统上下文"在每次AI请求时携带
- 文件存储在本地（本地模式）或上传至服务器（服务器模式）
- 每次会话可重新上传/清除背景文件

### 3.3 回答记录（会话管理）

#### 3.3.1 会话定义
- 一次**完整会话** = 从点击"开始回答"到点击"退出"之间的所有交互
- 会话包含：背景文件、所有问答对、时间戳

#### 3.3.2 历史记录功能
- 按时间倒序展示历史会话列表
- 每个会话显示：日期、问题数量、背景文件名称
- 支持搜索（按问题标题关键词）
- 支持导出（JSON/Markdown格式）
- 本地模式存储在SQLite，服务器模式存储在PostgreSQL

#### 3.3.3 数据隔离
- 本地API模式：数据完全存储在本地Electron应用内
- 服务器模式：数据存储在服务器，按用户ID隔离

### 3.4 知识库

#### 3.4.1 功能说明
知识库是一个独立的知识资料管理系统，用于存储回答者常用的参考资料。

#### 3.4.2 操作功能
| 功能 | 说明 |
|------|------|
| **创建条目** | 标题 + 内容（Markdown编辑器） |
| **上传文档** | 支持PDF/Word/TXT，自动提取文本 |
| **分类管理** | 自定义分类/标签 |
| **搜索** | 全文搜索知识库内容 |
| **编辑/删除** | 修改或删除已有条目 |
| **AI调用** | 回答时勾选知识库条目作为额外上下文 |

#### 3.4.3 数据结构
```
知识库条目 {
  id: string
  title: string
  content: string          // Markdown格式
  category: string
  tags: string[]
  sourceFile?: string      // 原始文件名
  createdAt: Date
  updatedAt: Date
}
```

### 3.5 联系我们与设置

#### 3.5.1 联系我们
- 显示联系渠道（邮箱、微信、GitHub Issue）
- 常见问题FAQ
- 反馈表单（本地模式下仅展示，服务器模式下可提交）

#### 3.5.2 设置页面
| 设置项 | 说明 |
|--------|------|
| **API配置** | API Key、Base URL（支持OpenAI/Claude/自定义） |
| **模型选择** | gpt-4o / gpt-4-turbo / claude-3.5-sonnet 等 |
| **运行模式** | 本地API / 服务器API 一键切换 |
| **服务器地址** | 服务器API的Base URL |
| **主题** | 浅色/深色/跟随系统 |
| **语言** | 中文/English |
| **快捷键** | 可自定义快捷键 |
| **数据导出** | 导出全部数据 |
| **自动更新** | 检查更新开关 |

## 四、用户交互流程

```
应用启动
  │
  ├─ 首次使用 ─→ 引导页 ─→ 配置API / 登录
  │
  └─ 已配置 ──→ 主界面
                  │
                  ├─ [知识库] ─→ 管理知识资料
                  │
                  ├─ [开始回答] ─→ 上传背景文件 ─→ 工作台
                  │                    │
                  │                    ├─ 接收问题 ─→ AI自动回答
                  │                    ├─ 查看历史问答
                  │                    └─ [退出] ─→ 保存会话 ─→ 返回主界面
                  │
                  ├─ [历史记录] ─→ 查看/搜索/导出历史会话
                  │
                  └─ [设置] ─→ 配置API/模式切换/联系我们
```

## 五、非功能需求

| 需求 | 指标 |
|------|------|
| **响应时间** | AI回答首字延迟 < 3秒 |
| **离线能力** | 本地API模式完全离线可用 |
| **数据安全** | 本地数据AES加密存储；服务器端HTTPS + JWT |
| **扩展性** | 客户端React代码可80%复用于React Native APP |
| **兼容性** | Windows 10+ / macOS 12+ / Linux (Ubuntu 20.04+) |
